BRIDGES: Changes in Version 3.0.0

  • Dataset: We now support accessing Maps through Open Street Map (see tutorial)
  • Dataset: We now support accessing Song Lyrics through Genius (see tutorial)
  • Dataset: We now support accessing much larger Movie/Actor data through Wikidata (see tutorial)
  • Interactive application: We now support basic games in Bridges. (see tutorial)
  • Charting Capabilities: We now support Line Charting capabilities. This can be helpful in performing timing experiments or to show temporal datasets. (see documentation.)
  • New Data Structure: We now support spatial datastructures such as KdTrees. (See tutorial.)
  • Drawing Capabilities: We now support shape collections to make simple drawings. (See tutorial
  • Revamping: In classes where it is appropriate, Bridges now use language constructs to make programming easier such as iterators, operator overloading, and properties. See documentation of specific classes to see what is supported.

BRIDGES: Changes in Version 2.3.3

  • GraphAdjList and GraphAdjMatrix now have 3 generic parameters, K for key type to access vertices and edges, E1, that stores vertex specific information and E2, to support edge specific information. Accessort methods to set/get vertex and edge data are provided.

    In the C++ version, E1 defaults K and E2 defaults to E1, thus, simple graphs (that dont care about vertex or edge attributes) can be declared with a single parameter for the key.

    Since Java does not support defaults for generic parameters, two classes GraphAdjListSimple, GraphAdjMatrixSimple are provided that provide the same functionality. These extend the graph classes with default values for the edge and vertex information types(currently set to String).

  • The graph classes provide convenience methods to easily set visual attributes for vertices and edges. getVisualizer(K v) and getLinkVisualizer(K v1, K v2) provide access to the visualizer objects that are then used to set any of the supported visual attributes.
  • Labels are now supported for graph edges. The label is defined as part of the LinkVisualizer object (setLabel()/getLabel()).
  • Faster rendering of large graphs (using Canvas)
